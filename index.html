In this experiment I want to see if I can build my own CSS styles using Open Props as a design system, but without the normalize and buttons opinionated styles of Open Props. Different pages of this site will demonstrate different styling options.

I decided to build the styles using a layered approach so that these layers can be used independently and stacked on top of each other as required.

Open-Props layer

The first layer is Open Props itself. It just declares useful properties that can be used in other layers for consistency. This layer does not make any visual changes to the HTML pages.

Normalize layer

The normalize layer is used as a base style to use for most of my internal projects. It should do a CSS reset and create basic styling for semantic HTML tags. This layer also offers light and dark themes dependent on the operating system's light/dark mode. It is also responsive based on the size of the device screen and resolution, to look good on anything from a large 4K display to a entry level mobile phone.

This layer will also set up print styles so that printing a page would default to the normalized view with some slight modifications. It should not print the body header and footer tags and only print the main tag.

Any layer built on top of the normalize layer should declare its styles inside a @media screen CSS selector so that it does not break the printable styles configured by the normalize layer.

LCARS layer

The LCARS layer is inspired by the Star trek LCARS user interface as seen on the popular TV series. I say inspired by as I did not want to go all the way and try to clone that look and feel. When viewed in light mode users may not even realize the inspiration, but some may realize it when viewed in dark mode.

The main purpose of this layer is to add some unique characted on top of the normalize layer as the normalize layer could be viewed as being a bit boring. It should show what is possible by the layered approach.
